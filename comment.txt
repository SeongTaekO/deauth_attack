pcap_t *pcap_open_live(const char *device, int snaplen, int promisc, int to_ms, char *errbuf);
device: 캡처할 네트워크 인터페이스의 이름을 나타내는 문자열입니다. 예를 들어, "eth0" 또는 "wlan0"과 같은 인터페이스 이름을 사용할 수 있습니다.
snaplen: 캡처된 패킷의 최대 길이를 지정하는 정수 값입니다. 이 값을 너무 작게 설정하면 패킷의 일부가 잘려서 나타날 수 있고, 너무 크게 설정하면 메모리를 많이 사용할 수 있습니다.
promisc: promiscuous 모드를 사용할지 여부를 나타내는 정수 값입니다. 0은 promiscuous 모드를 사용하지 않음을 의미하고, 1은 promiscuous 모드를 사용함을 의미합니다. promiscuous 모드는 네트워크 상의 모든 트래픽을 캡처하는 모드입니다.
to_ms: 캡처할 시간을 밀리초 단위로 지정하는 정수 값입니다. 0을 입력하면 무한대로 캡처를 계속합니다.
errbuf: 오류 메시지를 저장할 문자열 버퍼입니다.
반환 값:
성공적으로 네트워크 인터페이스를 열었을 경우에는 pcap_t 구조체에 대한 포인터를 반환합니다. 실패한 경우에는 NULL을 반환합니다.


%02x와 %02hhx는 둘 다 형식 지정자(format specifier)이며, C 프로그래밍 언어에서 주로 printf()나 scanf()와 같은 함수에서 사용됩니다. 하지만 두 지정자 간에는 약간의 차이가 있습니다.
%02x:
%02x는 두 자리의 16진수(hexadecimal)를 표현합니다.
앞에 0을 붙이고, 필요한 경우 빈 자리에 0을 채웁니다.
주로 정수형 변수를 16진수로 출력할 때 사용됩니다.
%02hhx:
%02hhx는 %02x와 유사하지만, 추가적으로 hh가 붙어 있습니다.
hh는 해당 변수가 char 자료형임을 명시하는 것으로, 1바이트 크기의 변수를 의미합니다.
이 형식 지정자는 주로 char 자료형의 변수를 16진수로 출력할 때 사용됩니다.
따라서 %02x는 정수형 변수를 16진수로, %02hhx는 char 자료형 변수를 16진수로 출력할 때 주로 사용됩니다. %02hhx는 추가로 변수의 크기를 명시하여 더 명확하게 지정합니다.
